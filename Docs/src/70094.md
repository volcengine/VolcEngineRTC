# 方法
## 引擎管理

| 方法 | 描述 |
| --- | --- |
| [createRTCVideo](70095.md#creatertcvideo) | 创建 IRTCVideo 实例。  <br><br/>如果当前进程中未创建引擎实例，那么你必须先使用此方法，以使用 RTC 提供的各种音视频能力。  <br><br/>如果当前进程中已创建了引擎实例，再次调用此方法时，会返回已创建的引擎实例。 |
| [destroyRTCVideo](70095.md#destroyrtcvideo) | 销毁由 [createRTCVideo](70095.md#creatertcvideo) 所创建的引擎实例，并释放所有相关资源。<br> |

| [feedback](70095.md#IRTCVideo-feedback) | 将用户反馈的问题上报到 RTC |
| [getErrorDescription](70095.md#geterrordescription) | 获取 SDK 内各种错误码、警告码的描述文字。 |

| [setRuntimeParameters](70095.md#IRTCVideo-setruntimeparameters) | 设置运行时的参数 |

## 房间管理

| 方法 | 描述 |
| --- | --- |
| [createRTCRoom](70095.md#IRTCVideo-creatertcroom) | 创建房间实例。<br><br/>调用此方法仅返回一个房间实例，你仍需调用 [joinRoom](70095.md#IRTCRoom-joinroom) 才能真正地创建/加入房间。<br><br/>多次调用此方法以创建多个 [IRTCRoom](70095.md#irtcroom) 实例。分别调用各 IRTCRoom 实例中的 [joinRoom](70095.md#IRTCRoom-joinroom) 方法，同时加入多个房间。<br><br/>多房间模式下，用户可以同时订阅各房间的音视频流。 |
| [destroy](70095.md#IRTCRoom-destroy) | 退出并销毁调用 [createRTCRoom](70095.md#IRTCVideo-creatertcroom) 所创建的房间实例。 |
| [setRTCRoomEventHandler](70095.md#IRTCRoom-setrtcroomeventhandler) | 通过设置 [IRTCRoom](70095.md#irtcroom) 对象的事件句柄，监听此对象对应的回调事件。 |
| [joinRoom](70095.md#IRTCRoom-joinroom) | 创建/加入房间：房间不存在时即创建房间；房间存在时，未加入房间的用户可加入这个房间。  <br><br/>同一房间内的用户间可以相互通话。  <br> |
| [setRoomExtraInfo](70095.md#IRTCRoom-setroomextrainfo) | 设置/更新房间附加信息，可用于标识房间状态或属性，或灵活实现各种业务逻辑。 |
| [leaveRoom](70095#IRTCRoom-leaveroom) | 离开房间。  <br><br/>用户调用此方法离开房间，结束通话过程，释放所有通话相关的资源。  <br><br/>加入房间后，必须调用此方法结束通话，否则无法开始下一次通话。无论当前是否在房间内，都可以调用此方法。重复调用此方法没有负面影响。  <br><br/>此方法是异步操作，调用返回时并没有真正退出房间。真正退出房间后，本地会收到 [onLeaveRoom](70096.md#IRTCRoomEventHandler-onleaveroom) 回调通知。  <br> |
| [setUserVisibility](70095.md#IRTCRoom-setuservisibility) | 设置用户可见性。未调用该接口前，本地用户默认对他人可见。 <br> |
| [updateToken](70095.md#IRTCRoom-updatetoken) | 更新 Token。<br/>Token 中同时包含进房、发布和订阅权限，各权限有一定的有效期，并且到期前 30 秒会触发回调，提示用户更新 Token 相关权限。此时需要重新获取 Token，并调用此方法更新 Token，以保证通话的正常进行。 |
